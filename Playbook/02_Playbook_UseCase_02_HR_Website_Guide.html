🌐 HR tự build website Sổ tay nhân sự bằng AI

Biến tài liệu rời rạc thành cổng thông tin dễ đọc, dễ tìm — HR non-tech có thể tự làm và duy trì

Owner: HR Team Trạng thái: Validated ✅ Công cụ: Mintlify / Claude / Antigravity / GitHub Cập nhật: 06/2026 Độ khó: Trung bình — cần theo đúng quy trình
S — SITUATION (BỐI CẢNH)

Tài liệu nhân sự rời rạc, khó tra cứu

  • Quy trình, chính sách, phúc lợi, hội nhập phân tán trên Drive, Slack, file nội bộ.
  • Nhân viên khó tìm kiếm thông tin nhanh chóng, thường xuyên phải hỏi lại HR.
  • Các văn bản mang tính chất "document-like", dài dòng, khó đọc lướt.
T — TASK (NHIỆM VỤ)

Tạo cổng thông tin (Employee Portal)

  • Xây dựng một website Sổ tay nhân sự duy nhất, dễ đọc, dễ tìm, dễ cập nhật.
  • Mục tiêu không phải làm website phức tạp, mà là hệ thống hóa tài liệu.
  • Nhân sự không chuyên về tech (non-tech) cũng có thể tự build và duy trì bằng AI.
A — ACTION (CÁCH LÀM)

Quy trình kết hợp nhiều AI Tools

  • Chốt cấu trúc (sitemap) và gom nội dung thô theo từng trang.
  • Dùng ChatGPT / Claude chuẩn hóa văn phong employee-friendly, chia section hợp lý.
  • Dùng Antigravity / Cursor để build layout từng trang trên nền tảng Mintlify.
  • Review, preview local, sau đó commit & push lên GitHub để tự động deploy.
R — RESULT (KẾT QUẢ)

Hệ thống tài liệu hiện đại, tự động

  • 1 portal duy nhất cho mọi thông tin HR với giao diện clean, chuẩn nhận diện.
  • Giảm thời gian support lặp lại của HR, tăng tỷ lệ self-serve của nhân viên.

⚙️ Chuẩn bị trước khi bắt đầu

Bước 0.1 — Tạo tài khoản Mintlify

  • Truy cập: mintlify.com → Nhấn Get Started → Đăng ký bằng tài khoản GitHub (khuyên dùng) hoặc email.
  • Tạo project: Sau khi đăng nhập, Mintlify sẽ hỏi tên project và màu sắc. Điền thông tin, nhấn Create.
  • Mintlify sẽ tự tạo: Một GitHub repo mẫu chứa file docs.json, index.mdx, và introduction.mdx — đây là điểm xuất phát của bạn.
  • Lưu ý: Gói Free của Mintlify cho phép 1 site public và không giới hạn số trang. Đủ dùng cho sổ tay nội bộ.

Bước 0.2 — Cài đặt GitHub Desktop

  • Tải về: desktop.github.com → Nhấn Download for Mac / Windows → Cài như app bình thường.
  • Đăng nhập: Mở GitHub Desktop → Sign in to GitHub.com → Điền email/mật khẩu tài khoản GitHub của bạn.
  • Clone repo về máy:
    • Vào File → Clone Repository → Chọn repo Mintlify vừa tạo ở bước 0.1.
    • Chọn thư mục lưu trên máy (ví dụ: Documents/HR-Handbook).
    • Nhấn Clone — GitHub Desktop sẽ tải toàn bộ file về máy.
  • Kết quả: Bạn có thể mở thư mục đó và thấy các file .mdx, docs.json — đây là nội dung website của bạn.

Bước 0.3 — Cài đặt Mintlify CLI (để preview local)

  • Yêu cầu trước: Cần cài Node.js trước. Tải tại nodejs.org → Chọn phiên bản LTS → Cài như app bình thường.
  • Cài Mintlify CLI: Mở Terminal (Mac) hoặc Command Prompt (Windows) → Dán lệnh sau và nhấn Enter:
    npm i -g mintlify
  • Chạy preview: Mở Terminal → điều hướng đến thư mục repo → gõ mintlify dev → Mở trình duyệt tại localhost:3000 để xem trước.
  • Nếu không muốn cài CLI: Bỏ qua bước này — push code lên GitHub là Mintlify sẽ tự deploy và bạn xem link live sau khoảng 1–2 phút.
  • Mẹo: Nhờ Antigravity hoặc Claude chạy lệnh này giúp nếu chưa quen với Terminal.

Bước 0.4 — Kết nối Mintlify với GitHub (Auto Deploy)

  • Mintlify tự kết nối: Nếu bạn đăng ký Mintlify bằng tài khoản GitHub (bước 0.1), repo Mintlify tạo sẽ được kết nối tự động — mỗi lần bạn push code lên nhánh main, Mintlify sẽ tự deploy.
  • Kiểm tra kết nối: Vào dashboard.mintlify.com → Chọn project → Tab Settings → Mục GitHub → Xem repo đã connect chưa.
  • Nếu chưa kết nối: Nhấn Connect GitHub → Chọn đúng repo → Authorize. Sau đó mọi lần push lên main sẽ tự deploy.
  • Xem link live: Vào Dashboard → Tab Overview → Copy link dạng ten-project.mintlify.app — đây là link public của website.

🛠️ Chi tiết quy trình 7 bước (The Workflow)

Bước 1 — Chốt cấu trúc website (Sitemap)

  • Làm gì: Liệt kê tất cả các trang/chủ đề cần có vào 1 file Google Doc. Ví dụ: Trang chủ, Hành trình hội nhập, Chính sách, Phúc lợi, AI Academy, Danh bạ hỗ trợ.
  • Cách làm: Nhóm các trang thành 3–5 hub lớn (mỗi hub = 1 nhóm chủ đề). Không cần nhiều hơn 5 hub nếu chưa cần thiết.
  • ⚠️ Quan trọng: Chốt sitemap trước khi bắt tay build. Sửa navigation sau khi đã build rất tốn thời gian và dễ bị rối.

Bước 2 — Gom nội dung thô theo từng trang

  • Làm gì: Với mỗi trang trong sitemap, copy toàn bộ nội dung thô (từ Drive, Confluence, Slack, email...) vào 1 chỗ. Chưa cần viết lại, chỉ cần gom đủ.
  • Format gom nội dung cho mỗi trang gồm: Tên trang + mục tiêu · Đối tượng đọc · Nội dung chính · Link / hình ảnh / biểu mẫu · Người phụ trách cập nhật.
  • Mẹo: Đừng lo nội dung chưa đẹp hay chưa gọn — AI sẽ giúp xử lý ở bước tiếp theo.
📂 Ví dụ nội dung thô đã gom — Ahamove Handbook

Bước 3 — Dùng Claude / ChatGPT chuẩn hóa nội dung

  • Làm gì: Paste nội dung thô của từng trang vào Claude hoặc ChatGPT, nhờ AI rút gọn, bỏ trùng lặp và viết lại theo văn phong employee-friendly.
  • Prompt mẫu: "Đây là nội dung thô cho trang [tên trang]. Hãy rút gọn, bỏ phần trùng lặp, chia thành các section ngắn, và viết lại theo văn phong thân thiện, dễ đọc cho nhân viên. Giữ nguyên tất cả thông tin quan trọng."
  • Output cần có: Nội dung đã gọn, đủ ý, sẵn sàng để đưa vào AI build ở bước tiếp theo.
📂 Ví dụ trang đã được chuẩn hóa

Bước 4 — Build layout bằng AI

  • Có thể dùng tool nào? Có 2 lựa chọn — cả hai đều được:
🤖 Antigravity (IDE) — kết nối trực tiếp với file repo, build & chỉnh sửa file MDX tại chỗ, phù hợp khi cần kiểm soát chi tiết từng file. 💬 Claude Projects (claude.ai) — không cần IDE, chat trên web, upload file MDX → Claude viết lại → copy paste lại. Phù hợp khi không cài app hoặc mới bắt đầu.
  • Thứ tự build khuyên dùng: docs.json (sitemap) → Trang chủ → Hành trình hội nhập → Các trang policy → AI Academy → Review layout toàn bộ & Fix links → Deploy.
  • Nguyên tắc prompt (cho cả 2 tool): Gửi từng cụm nhỏ (1–2 trang/lần). Mỗi prompt ghi rõ 4 phần: (1) Trang cần build, (2) Nội dung đã chuẩn bị, (3) Yêu cầu layout, (4) Những gì không được thay đổi.
  • Mẹo: Sau mỗi cụm, yêu cầu AI trả lại danh sách file đã thay đổi để dễ kiểm soát tiến độ.
📂 Ví dụ các trang đã được build — Ahamove Handbook
index.mdx (Trang chủ) day-1.mdx week-1.mdx first-month.mdx internal-systems.mdx payroll.mdx health-check.mdx ai-playbook.mdx

Bước 5 — Preview local trước khi đưa lên

  • Làm gì: Chạy lệnh mintlify dev trong Terminal để xem trước website trên máy tính tại localhost:3000. (Nhờ Antigravity / Claude chạy giúp nếu chưa quen với Terminal.)
  • Nếu dùng Claude Projects: Bước này bỏ qua được — upload file lên Claude, nhờ Claude preview bằng cách mô tả layout, hoặc dùng thẳng link live sau khi push.
  • Checklist review — kiểm tra 5 điều này:
    • ✅ Menu điều hướng đúng thứ tự, đúng tên trang chưa?
    • ✅ Có trang nào bị lỗi (trang trắng / link hỏng) không?
    • ✅ Nội dung có quá nhiều bullet list / card lặp lại không?
    • ✅ Xem thử trên điện thoại có đọc được không?
    • ✅ Gõ từ khóa vào ô Search có ra kết quả đúng không?
  • Nếu cần sửa: Mô tả vấn đề cụ thể cho AI (Antigravity hoặc Claude) và yêu cầu fix. Không tự sửa file code tay nếu chưa quen.

Bước 6 — Commit & Push lên GitHub

  • Làm gì: Dùng GitHub Desktop (ứng dụng có giao diện đồ họa, không cần gõ lệnh) để lưu lại thay đổi và đẩy code lên GitHub.
  • Nếu dùng Claude Projects: Sau khi Claude viết xong file MDX → Copy nội dung → Paste vào đúng file trong VS Code hoặc trình soạn thảo bất kỳ → Lưu lại → Tiếp tục dùng GitHub Desktop như bình thường.
  • Các bước thực hiện trong GitHub Desktop:
    • Mở GitHub Desktop → Danh sách file đã thay đổi hiển thị bên trái.
    • Điền tên commit ngắn gọn vào ô "Summary", ví dụ: Update HR handbook structure.
    • Nhấn Commit to main → Tiếp theo nhấn Push origin.
  • Lưu ý: Luôn push lên nhánh main. Mintlify sẽ tự động bắt đầu deploy ngay sau khi push thành công.
📂 Ví dụ các file trong cùng 1 commit
docs.json style.css offboarding.mdx org-structure.mdx

Bước 7 — Kiểm tra link live & Thông báo nội bộ

  • Làm gì: Sau 1–3 phút, mở link website thật (Mintlify cung cấp link public) để kiểm tra kết quả deploy.
  • Checklist cuối:
    • ✅ Mở link trên trình duyệt thường (không phải localhost) — xem đã lên chưa.
    • ✅ Nhấn Ctrl+Shift+R (hard refresh) nếu thấy nội dung cũ.
    • ✅ Kiểm tra nhanh 2–3 trang quan trọng nhất.
    • ✅ Forward link cho team hoặc đăng Slack thông báo.
  • Cập nhật sau này: Muốn cập nhật nội dung → Sửa bằng Antigravity → Preview (bước 5) → Commit & Push (bước 6) → Xong. Không cần làm lại từ đầu.

💡 Tips & Nguyên tắc thiết kế

  • Không nói chung chung với AI: Thay vì "làm trang này đẹp hơn", hãy yêu cầu cụ thể "chuyển phần lưu ý thành card grid, giảm danh sách".
  • Luôn ghi rõ 4 điều trong prompt: 1. Trang cần sửa; 2. Vấn đề hiện tại; 3. Yêu cầu chỉnh sửa; 4. Những gì không được đổi. (Luôn yêu cầu AI trả lại summary).
  • Cấu trúc rõ ràng: Một trang = một mục đích rõ ràng. Một section = một nhóm thông tin. Không lặp lại cùng một ý ở nhiều hub.
  • Nên dùng: Hero card ngắn gọn, timeline (nếu có quy trình), contact card (danh bạ hỗ trợ), table (khi so sánh).
  • Không nên dùng: Paragraph dài, bullet list quá nhiều, card lặp liên tục, nhiều section nhỏ lắt nhắt.
  • Deploy Checklist: Kiểm tra Navigation, không broken link, không nội dung trùng, email format đồng bộ, search hoạt động. Tắt Dark Mode bằng cách Force Light Theme.

🤖 Dùng AI để tạo prompt cho chính mình

Không cần tự nghĩ ra prompt từ đầu. Hãy nhờ chính AI viết prompt cho bạn. Cách đơn giản nhất: mô tả bạn đang làm gì và bạn muốn gì, sau đó thêm câu „Hãy viết cho tôi một prompt tốt để yêu cầu AI khác làm việc này.‟ Claude sẽ trả ra prompt chuẩn hơn bất kỳ thứ bạn tự nghĩ.

Nhưng nếu bạn muốn bắt đầu ngay, dưới đây là các prompt mẫu theo từng giai đoạn — copy và điền vào phần in nghiêng là dùng được ngay:

🟠 Khởi đầu — Nhờ AI tạo Prompt cho bạn
Tôi đang chuẩn bị build một website sổ tay nhân sự bằng Mintlify cho công ty [tên công ty].
Các nội dung cần có: [liệt kê các hub chính: hội nhập / chính sách / phúc lợi / hỗ trợ...].
Target audience: nhân viên nội bộ, không phải kỹ thuật.

Hãy giúp tôi:
1. Lên sitemap (tổng hợp các hub và trang con)
2. Gợi ý thứ tự build hợp lý
3. Viết cho tôi prompt mẫu để dùng với Antigravity / Cursor khi build từng trang
🟡 Bước 1 — Build docs.json (Sitemap)
Tôi cần tạo file docs.json cho một website Mintlify.

Cấu trúc dự kiến gồm các tab chính:
- Tab 1: Trang chủ (1 trang duy nhất)
- Tab 2: Hành trình hội nhập (giới thiệu công ty, ngày đầu tiên, phúc lợi)
- Tab 3: Quy trình hành chính (nghỉ phép, lương, hệ thống)
- Tab 4: Hỗ trợ (liên hệ, FAQ)

Màu theme của tôi: primary [mã màu]. Tên website: [tên].
Hãy tạo file docs.json hoàn chỉnh theo cấu trúc này, với các field: theme, name, colors, navigation, footer.
🟢 Bước 2 — Build trang Homepage
Tạo file index.mdx cho trang chủ của website nhân sự [tên công ty].

Nhiệm vụ của trang này: giới thiệu portal và định hướng người dùng đến đúng hub mình cần.
Layout cần có:
- Hero section ngắn gọn (tiêu đề + mô tả 1 dòng)
- Grid 4–5 card trỏ đến các hub chính (mỗi card: icon, tên hub, 1 dòng mô tả)
- Banner thông báo hoặc What’s New (tùy chọn)

Dùng Mintlify MDX component. Không dùng HTML thuần.
🟣 Bước 3 — Build trang nội dung từ tài liệu thô
Tạo file [tên-trang.mdx] cho trang „[tên trang]‟.

Mục đích trang: [mô tả ngắn]
Đối tượng đọc: [đối tượng]
Nội dung thô cần transform:
———
[paste nội dung thô vào đây]
———
Yêu cầu:
- Layout: hero card ngắn, section rõ ràng, ít bullet list
- Tối đa 1 table nếu có so sánh
- Cuối trang: 1 contact card hoặc link đến trang liên quan
- Không đặt thêm nội dung mới, chỉ transform nội dung thô đã cho
🔵 Review — QA trước khi Deploy
Hãy review toàn bộ các file trong folder này trước khi deploy.

Kiểm tra:
1. Navigation: các link trong docs.json có trỏ đúng file không?
2. Nội dung trùng: có trang nào lặp lại cùng một ý không?
3. Layout: có trang nào quá nhiều bullet list / card không?
4. Broken links: link internal có hoạt động không?
5. Mobile: có thành phần nào bị vỡ layout trên màn hình nhỏ không?

Trả lại:
- Danh sách lỗi cụ thể
- Gợi ý sửa
- Các file cần cập nhật
